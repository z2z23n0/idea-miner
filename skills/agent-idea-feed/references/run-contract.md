# Run Contract

## 来源访问

- 使用当前环境可用的网页搜索、浏览器或来源工具。
- 站内搜索被 Cloudflare、登录墙、动态加载或索引问题阻挡时，改用搜索引擎索引页、`site:` 查询、公开详情页、分类页和分页结果。
- Upwork 优先链接具体 job。
- Freelancer、PeoplePerHour、电鸭、实现网和猪八戒链接具体项目、需求或服务描述页。
- chinese-independent-developer 与六个平台平级。扫描主列表、程序员版、近期提交或产品链接，最终优先链接具体产品或仓库。
- 不绕过访问控制，不编造页面内容。不要提前停止；轮换核心词、分页和公开索引，直到每个来源找到三个合格结果。

## 提取方式

把招聘标题、项目标题或产品宣传改写成自然中文。重点写 Agent 替人完成的工作，不照抄原文，也不介绍承包方需要什么技术。

完整循环：

```text
用户给 Agent 什么
Agent 自己观察、判断和执行什么
用户最后得到什么结果
```

例如，原始需求如果描述“接收多个住宿平台消息、识别房间并发送入住信息”，可以写成：

```text
民宿值班经理 Agent：同时盯住住宿平台和聊天工具，回复住客、发送对应房间的入住信息、处理现场问题并完成新的预订安排。
```

不要把它改成消息路由评测器、酒店客服 benchmark 或多平台 Agent 观测台。

## 去重

开始前运行：

```bash
node scripts/seen-store.mjs list
```

去重不只比较标题和 URL，还要比较完整任务。只要目标用户、触发方式、主要行动和最终结果基本相同，就视为重复，即使名称、行业词或来源不同也不能再次输出。

最终结果写成临时 JSON：

```json
{
  "run_id": "run_YYYY-MM-DD_HHmm",
  "ideas": [
    {
      "platform": "Upwork",
      "title": "…… Agent",
      "description": "它替用户完成什么完整任务。",
      "url": "https://..."
    }
  ]
}
```

然后执行：

```bash
node scripts/seen-store.mjs record /path/to/run.json
```

`record` 会检查：恰好 21 条、七个来源各 3 条、字段完整、运行内无重复、没有命中历史记录。成功后再返回最终结果。
